Read this before drawing conclusions:
  • SIC industry codes are coarse and dated — a starting axis, not ground truth. "SIC 35" spans farm machinery, HVAC, and computer hardware alike.
  • Percentile is a company's position within its peer group, not a good/bad verdict — for some metrics a higher value is not "better."
  • Ranks exclude peers for which a metric is N/A — an N/A company is never counted as a low value.
  • Named-peer bars use only companies with a resolvable value for this exact metric/period — not every company reports on a comparable fiscal calendar.
  • "FY2023" means each company's own fiscal year, not the calendar year — the named-peer companies' fiscal year-ends span Jun 2023 to Dec 2023 (see each bar's FYE date), so these are close-in-time, not perfectly aligned snapshots.
  • Trend sparklines (5 fiscal years, 2021–2025) and the FY2023 peer percentile are on different time bases — the percentile applies only to FY2023, not to the years shown in the trend line.
  • The gross-margin and revenue-growth comparisons use a different peer set than the net-margin one above (large-cap tech, not strictly SIC-35) — most SIC-35 peers in this dataset were only ingested via the cross-company screening backfill (6 concepts: revenue, net income, total assets/liabilities, equity, cash), which isn't enough data to compute gross margin or YoY growth for them. See the note below on why only 3 metrics currently have peer-percentile coverage.
  • Everything from "Quarterly health trends" through "Peer value distribution" uses a different, synthetic 12-company peer universe (Apple's real CIK/SIC plus 11 generated peers), run through the real percentile/z-score/distribution jobs — genuine pipeline math, but not the same ingest as the FY2023 sections above, and not Apple's actual reported financials. See the data-note below.
  • The "Ownership signals" section (insider activity, institutional trend, 13D/G holders) is fully illustrative — hand-authored numbers, not output from a real Form 4 or 13F ingest. It is not transcribed from any pipeline run, unlike every other section on this page.
  • Insider "net shares disposed" is a raw count of reported Form 4 transaction codes, not a sentiment signal — most large-cap insider selling is routine, scheduled 10b5-1 activity, not a reaction to company news.
  • Institutional ownership (13F) is a long-only, quarter-end snapshot filed up to 45 days late — it never reflects short positions or the most recent 6–7 weeks of trading.
Why only 3 metrics have peer-percentile coverage (net margin, ROA, asset turnover):

Of the 6,735 companies in this dataset, 6,721 (99.8%) were ingested only through the cross-company screening backfill (Milestone 4), which pulls exactly six SEC "frames" concepts per company — revenue, net income, total assets, total liabilities, stockholders' equity, and cash — not a full filing history. Only 14 companies (Apple, Microsoft, Amazon, NVIDIA, Alphabet, Meta, Tesla, JPMorgan, Bank of America, Walmart, Costco, Target, PayPal, ASML) have a complete companyfacts ingest. Net margin, ROA, and asset turnover are the only three metrics computable purely from those six frames concepts — every other metric (growth rates, leverage, cash flow, per-share) needs a tag outside that set (e.g. cost of revenue, debt, share count), so it's N/A for the other 99.8% and never reaches the minimum peer-group size. This is a real, verified data-coverage gap, not a bug in the ranking math — fixing it means running the full companyfacts backfill across the peer universe, not adjusting peer_ranks.py.

Where the quarterly / percentile-trajectory / distribution numbers come from:

There's no production database in this repo, and a real SEC bulk companyfacts backfill needs network access and hours of runtime — out of scope for authoring one static example page. So scripts/seed_infographic_fixture.py builds a synthetic-but-structurally-real 12-company SIC-35 peer universe (Apple's real CIK 320193 and SIC 3571, plus 11 generated peers spread across margin/leverage/turnover profiles) with 12 quarters of full concept coverage, then runs the unmodified production jobs end to end — metrics_backfill → peer_ranks → peer_distribution — against it. Every number in those three sections is transcribed from that pipeline's real output, the same "hand-copied from a real response" pattern used for the FY2023 sections above, just from a richer fixture instead of live ingested data. The flat-line appearance of several metrics (net margin, gross margin, debt/equity, the 72.7th-percentile trajectory) is a genuine property of this fixture's generator, not a rendering artifact — it uses constant per-company ratios rather than a noisy quarter-to-quarter model.
