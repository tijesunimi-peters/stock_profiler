# tracks/ — parallel launch-readiness work logs

One notes file per track agent (`code.md`, `data.md`, `writing.md`, `infra.md`),
created by the agent on first run. Dated, append-only progress notes with evidence
(counts, test output, file paths) — the raw material the `/launch-parallel`
orchestrator uses to flip checkboxes in `../LAUNCH_READINESS.md`.

Rules: each agent writes ONLY its own file; the orchestrator owns
`LAUNCH_READINESS.md`; nothing here is public-facing copy.
